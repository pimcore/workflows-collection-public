# Pimcore Solution Architect (aka Pimcore Maestro)

## Who they are
IT affine users with technical understanding who configure and manage Pimcore environments.

## Goals and Responsibilities
- Define data models, permissions, and workflows.
- Configure imports and exports, perspectives, and languages.
- Manage portals and system-wide settings.
- Samples for configuration tasks:
  - Configuring users, roles, and permissions
  - Managing perspectives
  - Configuring data model (classes, predefined properties, field collection, object bricks….)
  - Setting up and configuring workflows
  - Datahub configurations
  - Setting up custom reports
  - Setting up dashboards
  - Setting up translation providers
  - Setting up copilot actions
  - Setting up portals
  - Configure redirects
  - Manage marketing settings

## Skill set
- Business Domain Understanding
  - Understand the customer’s business processes and data requirements.
  - Gasp customer’s data model requirements and data flows.
  - Translate requirements into Pimcore configurations.
- Data Modelling
  - Understand core concepts of data modelling like entities, data fields and their data types, and relationships.
  - Read and interpret UML or entity diagrams.
- Data Integration and Flows
  - Understand APIs and interface concepts:
    - Work with REST, GraphQL, and Webhooks.
  - Handle file-based data exchange:
    - Read and understand JSON, XML, and CSV files.
    - Understand file encodings and their effects
    - Know how to validate file structures and contents.
  - Understand integration patterns:
    - Push vs pull.
    - Synchronous vs asynchronous communication.
    - Different file transfer protocols like HTTP(s), sFTP.
  - Understand data transformation concepts:
    - Identify and interpret source data structures.
    - Understand needed target formats.
    - Use transformation operators to modify and map data.
  - Work with no-code/low-code automation:
    - Create and adjust automation workflows using tools like n8n.
- Configuration and Expressions
  - Be able to read, understand, and modify YAML configuration files.
  - Use Symfony Expression language:
    - Apply logical operators (AND, OR, NOT, parentheses).
    - Understand object references, attributes, and method calls in expressions.
  - Use Twig language for templating and defining custom output formats (e.g. dynamic messages)
  - Be able to read, write, and understand PQL (Pimcore Query Language) statements.
  - Understand and use regular expressions (regex):
    - Know what regex is and its use cases and be able to read and write basic regex patterns.
    - Use tools and resources (e.g., regex testers, AI assistants) to create and validate more complex expressions, even without understanding every detail.
- Environments and Deployments
  - Understand deployment workflows in Pimcore PaaS setups:
    - Branching for test environments.
    - Merging changes and deploying.
- Infrastructure Awareness
  - Know the roles of system components:
    - Database, application server, web server, background workers, queues, indexing engine, caching.
  - Understand data flows between these components and their implications in production environments (e.g., indexing delays, async processes).
  - No deep configuration, operations or maintenance knowledge required.
- Documentation and Communication
  - Understand technical documentation.
  - Communicate clearly about system behavior and data flows.
- Pimcore-Specific Knowledge
  - Understand Pimcore fundamentals, UI concepts and data architecture.
  - Know Pimcore basic data types documents, assets, data objects and their use cases.
  - Setup users and permissions with role-based permission system.
  - Know and configure all needed Pimcore features using documentation and academy.
